# 英语听力学习 SPA 改造任务

## 背景

当前项目是一个静态餐厅落地页。目标是改造成英语听力学习 App 原型，覆盖手机和 iPad 布局，核心遵循“精听打底 + 跟读练嘴 + 可理解输入 + 听写暴露盲区 + 听力词汇复习”的训练路线。

## 产品原则

- 首页直接进入学习工作台，不做营销型介绍页。
- 训练流程围绕“听一句 -> 复述/跟读 -> 对照原文 -> 标记盲区 -> 复习”闭环。
- 只使用略低于或略高于当前水平的可理解材料，避免把高难材料当噪音输入。
- 字幕默认可隐藏，声音优先于文字。
- 用户个人数据本地优先，第一版不做账号和云同步。
- 路由切换模拟 iOS 原生 App 的 push/pop/tab 切换效果，动画克制、快速、服务学习状态。

## 技术范围

- 单页路由应用：使用 hash route，支持深链和刷新恢复。
- 数据层：课程材料放在 `data/lessons.json`。
- 个人数据：使用 IndexedDB 保存进度、训练记录、词汇卡和设置。
- 动画：使用 CSS transition 与 Web Animations API 实现 iOS 风格页面切换和局部状态反馈。
- 响应式：手机使用底部 Tab，iPad/桌面使用左侧导航 + 主工作区 + 右侧详情栏。
- 静态部署：保持无需构建即可部署到 GitHub Pages。

## 第一版路由

- `#/today`：今日训练、连续学习、推荐材料。
- `#/train/:lessonId`：精听/跟读训练播放器。
- `#/dictation/:lessonId`：听写模式。
- `#/library`：素材库。
- `#/vocab`：听力词汇复习。
- `#/stats`：盲区报告。
- `#/settings`：训练偏好。

## 实现任务

- [x] 1. 建立静态 SPA 文件结构
  - 文件：`index.html`、`styles/app.css`、`scripts/app.js`、`data/lessons.json`
  - 目的：替换餐厅落地页，形成离线优先学习工具基础。

- [x] 2. 实现路由和 iOS 风格转场
  - 支持 tab 切换、详情 push、返回 pop。
  - 页面切换使用短时 slide/fade，动画时长控制在 160-260ms。
  - 避免动画影响正文可读性和输入体验。

- [x] 3. 实现 IndexedDB 数据层
  - 保存 lesson progress、attempts、mistakes、vocabCards、settings。
  - 首次打开自动初始化默认设置和示例词汇卡。

- [x] 4. 实现今日训练和素材库
  - 根据 JSON 材料展示今日推荐、训练目标和材料属性。
  - 素材卡展示等级、口音、时长、可懂度、推荐模式。

- [x] 5. 实现训练播放器
  - 支持逐句切换、隐藏/显示原文、循环句子、语速切换、模式切换。
  - 支持标记“连读、弱读、生词、口音、语速快”等听力盲区。
  - 训练行为写入 IndexedDB。

- [x] 6. 实现听写、词汇、统计和设置
  - 听写页支持输入、对照、错词高亮。
  - 词汇页使用“先听音再识义”的卡片形式。
  - 统计页按错误类型展示盲区分布。
  - 设置页保存每日目标、默认语速和字幕偏好。

- [x] 7. 验证
  - 本地静态服务打开无控制台阻塞错误。
  - 路由切换、训练交互和 IndexedDB 写入可用。
  - 手机宽度与 iPad/桌面宽度布局不重叠、不溢出。
